### S.O.L.I.D.

### Clean Code Principals

- [S]ingle Responsibility (SR):

  1.  Every class should have only one responsibility.
  2.  Reduce all your components to that single responsibility.

- [O]pen-Closed (OC):

  1.  Software entitites should be open for extension but closed for modification.
  2.  Code in a way so that your component is easily extensible without modification.

- [L]iskov Substitution (LS):

  1.  SubType objects should be substitutable for SuperType objects.

- [I]nterface Segregation (IS):

  1.  Clients should not depend on Interfaces that they do not use.

- [D]ependency Inversion (DI):

  1. An entity should depend upon abstractions and not concretions.
